# Summaries

This folder contains the completion summary per task.

Written by the planner after the implementation is reviewed and the task is closed.

## File

```text
summaries/<task-id>.md
```

## Required Structure

```md
# <task-id>

Owner: planner | reviewer
Status: done | blocked

## Delivered

## Not Delivered

## Tests

## Review Status

## Remaining Risks

## Recommended Next Steps
```

## Rules

- Only document real results
- Name tests specifically, don't make blanket claims
- If no implementation review happened, state that explicitly
- Document who actually implemented
- Only list follow-up work that genuinely remains
